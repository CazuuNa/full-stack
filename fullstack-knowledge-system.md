## 1. 后端工程能力（核心）

### 1.1 Node.js 运行时深入

- **核心概念**
  - Event Loop、宏任务/微任务、异步 I/O
  - 内存模型与 GC、内存泄漏常见来源
  - 进程与线程：Cluster、Worker Threads、child_process
- **关键能力**
  - 能解释“为什么某段代码阻塞/吞吐下降”
  - 能定位 CPU 飙升、内存上涨、句柄泄漏（连接/定时器）
  - 能做基本性能分析（火焰图/heap snapshot）
- **常用工具**
  - Node --inspect、Chrome DevTools、clinic.js（如项目已采用再深入）
  - pino/winston（日志库，择一）
- **验收产出**
  - 输出一份 Node 服务的性能基线：QPS、P95、内存占用与 GC 情况

### 1.2 Web 框架与工程化（Express/Fastify/NestJS）

- **核心概念**
  - 路由、控制器、服务层、数据访问层（Repository/DAO）
  - 中间件/拦截器（日志、鉴权、限流、错误处理）
  - 依赖注入（DI）、模块化、配置分环境（dev/stage/prod）
- **关键能力**
  - 能搭建一个可扩展骨架：目录结构、模块边界、统一异常、参数校验
  - 能把“业务逻辑”与“HTTP 细节”隔离，保证可测试性
- **重要组成部分（建议形成统一规范）**
  - 请求校验：DTO/schema（zod/joi/class-validator 选型与落地）
  - 错误体系：业务错误码、HTTP 映射、全局异常处理
  - 响应标准：分页、列表过滤、排序、版本管理（/v1）
  - 配置管理：环境变量 + schema 校验 + 配置热更新策略
- **验收产出**
  - 一套“脚手架级”后端模板：鉴权、日志、错误码、Swagger/OpenAPI

### 1.3 API 设计（REST / GraphQL / gRPC）

- **REST（必修）**
  - 资源建模、HTTP 动词语义、状态码、幂等性设计（PUT/DELETE）
  - 分页（offset/cursor）、过滤/排序、字段裁剪（可选）
  - API 版本策略（URL/Header）、兼容性与弃用策略
- **GraphQL（选修）**
  - Schema、Resolver、N+1 问题与 DataLoader 思路
  - 授权（字段级权限）与查询复杂度限制
- **gRPC（了解）**
  - Proto、IDL、流式请求、服务间高效通信场景
- **重要组成部分**
  - OpenAPI/Swagger 文档、契约测试、Mock、接口评审流程
- **验收产出**
  - 输出 OpenAPI 文档 + 生成 SDK（可选）+ 接口用例（Postman/Insomnia）

### 1.4 分层架构与领域建模（从“能写”到“可维护”）

- **核心概念**
  - 分层：Controller / Service / Domain / Repository
  - DDD 基础：实体、值对象、聚合根、领域服务、领域事件（了解）
  - 依赖倒置：上层依赖抽象（接口），便于测试与替换实现
- **关键能力**
  - 能明确模块边界：用户、订单、支付、库存等领域拆分（按业务）
  - 能识别“领域规则”并集中维护，减少散落 if-else
- **重要组成部分**
  - 统一返回结构、统一异常与错误码
  - 防腐层（Anti-Corruption Layer）：对第三方系统封装
- **验收产出**
  - 业务模块具备清晰边界与测试覆盖（Service/Domain 层）

### 1.5 数据访问、事务与并发控制

- **核心概念**
  - ORM/Query Builder：模型映射、迁移（migration）、连接池
  - 事务：ACID、隔离级别、幻读/不可重复读、死锁
  - 并发控制：乐观锁（version）、悲观锁（for update）
- **关键能力**
  - 能写出正确的事务边界：哪些逻辑必须同事务、哪些应拆分
  - 能定位慢查询/锁等待，并做索引与 SQL 优化
- **重要组成部分**
  - 数据库迁移规范：可回滚、可重复执行、按环境管理
  - 连接池配置：最大连接、超时、重试策略
- **验收产出**
  - 一个典型“扣库存/下单”流程：并发正确、无超卖、可回滚

### 1.6 鉴权与权限（上线必备）

- **认证（Authentication）**
  - Session/Cookie、JWT、Refresh Token、Token 失效策略
  - OAuth2/OIDC：授权码模式、PKCE、单点登录（SSO）
- **授权（Authorization）**
  - RBAC：用户-角色-权限
  - ABAC：基于属性的策略（了解）
  - 多租户：租户隔离策略（库/表/字段/权限层）
- **重要组成部分**
  - 权限校验落点：路由级、资源级、字段级（按产品复杂度选择）
  - 审计：谁在何时做了什么（操作日志）
- **验收产出**
  - 一套可扩展权限模型 + 管理端配置能力（最小可用版也行）

### 1.7 安全（必修）

- **Web 安全基础**
  - XSS、CSRF、SQL 注入、SSRF、路径穿越、命令注入
  - CORS、CSP、SameSite Cookie、HTTPS/TLS
- **数据与密钥安全**
  - 密码哈希：bcrypt/argon2（择一）
  - 加密与签名：HMAC、RSA/ECDSA（理解用途与边界）
  - Secrets 管理：不入库明文、不打印日志、环境变量与密钥管理服务（了解）
- **重要组成部分**
  - 输入校验与输出编码策略
  - 安全头：HSTS、X-Content-Type-Options、X-Frame-Options
  - 依赖安全：漏洞扫描与升级策略
- **验收产出**
  - 安全基线清单 + 安全测试用例（至少覆盖常见漏洞）

### 1.8 可靠性与工程质量

- **核心概念**
  - 幂等：去重键、业务幂等表、幂等 Token
  - 限流：固定窗口/滑动窗口/令牌桶/漏桶
  - 超时、重试（指数退避）、熔断、降级
- **重要组成部分**
  - 统一日志规范：请求 id/trace id、用户 id、业务关键字段
  - 错误码与告警：可定位、可统计、可回溯
  - 后台任务：定时任务、延迟任务、补偿任务
- **验收产出**
  - 在接口高并发下：稳定输出（错误可控、无雪崩、可快速定位）

## 2. 数据库与存储（核心）

### 2.1 关系型数据库（MySQL/PostgreSQL，必修）

- **核心概念**
  - 表设计：主键、外键、唯一约束、索引、范式与反范式
  - 索引：B+Tree、联合索引最左前缀、覆盖索引、回表
  - SQL 优化：执行计划、统计信息、慢查询
  - 事务与锁：行锁/间隙锁（MySQL）、MVCC
- **关键能力**
  - 能用执行计划解释性能问题并给出改法（索引/SQL/表结构）
  - 能设计“可演进”的表结构与迁移方案
- **重要组成部分**
  - 慢查询治理：阈值、采样、归因、回归测试
  - 备份恢复：全量/增量、RPO/RTO 概念
- **验收产出**
  - 一套核心业务表 + 索引设计说明 + 慢查询优化记录

### 2.2 Redis（必修，按工程化落地）

- **核心概念**
  - 常用数据结构：String/Hash/List/Set/ZSet、Bitmap、HyperLogLog（了解）
  - 过期与淘汰策略、持久化（RDB/AOF）、主从与哨兵（了解）
- **关键能力**
  - 缓存策略：Cache Aside、写穿/回写（了解）
  - 缓存问题：穿透/击穿/雪崩治理
  - 分布式锁：正确加锁/续期/解锁、避免误删
- **重要组成部分**
  - 热点 key 治理：分片、二级缓存、预热、限流
  - 数据一致性：延迟双删、消息通知/订阅（按系统复杂度选择）
- **验收产出**
  - 缓存命中率指标 + 缓存一致性方案说明 + 压测数据

### 2.3 搜索与分析（选修：ES / ClickHouse 等）

- **ElasticSearch**
  - 倒排索引、分词、mapping、聚合、索引生命周期（ILM）
  - 查询性能：filter vs query、分页深度问题（search_after）
- **适用场景**
  - 站内搜索、日志检索、复杂筛选与聚合
- **验收产出**
  - 一个可用的搜索服务：索引构建、增量更新、查询接口

### 2.4 对象存储与文件服务（必修到可落地）

- **核心概念**
  - 对象存储：Bucket/Object、ACL、生命周期、版本控制（了解）
  - 上传策略：服务端转发 vs 前端直传（签名 URL）
- **重要组成部分**
  - 大文件：分片/断点续传、秒传（可选）
  - CDN：缓存策略、回源、鉴权（基础）
- **验收产出**
  - 文件上传下载服务：权限控制、审计、可扩展的存储适配层

### 3.3 微服务与服务治理（按团队形态选择）

## 4. DevOps 与部署交付（核心）

### 4.1 Linux 基础（必修）

- **核心概念**
  - 文件系统与权限、进程、端口、日志与服务管理
  - 常用排障：网络连通、DNS、证书、磁盘、内存、CPU
- **重要组成部分**
  - 线上排障习惯：先观测后操作、记录变更、可回滚
- **验收产出**
  - 能在 Linux 环境独立部署与排障一套服务（含数据库/缓存）

### 4.2 Docker（必修）

- **核心概念**
  - 镜像分层、构建缓存、运行时网络、卷（volume）
  - 多阶段构建、非 root 运行、健康检查（healthcheck）
- **重要组成部分**
  - docker-compose：一键启动本地依赖（db/redis/mq）
  - 制品管理：镜像版本、回滚策略
- **验收产出**
  - 前后端 + 数据库/Redis 一键启动（本地与线上一致）

### 4.3 Nginx 与网关（必修）

- **核心概念**
  - 反向代理、负载均衡、静态资源、缓存、限速
  - HTTPS/TLS、证书续期、HTTP/2
- **验收产出**
  - 通过 Nginx 完成：域名、HTTPS、反代、静态资源缓存

### 4.4 CI/CD（必修）

- **核心概念**
  - 流水线阶段：Lint/Test/Build/Image/Deploy
  - 环境隔离：dev/stage/prod；配置注入与密钥管理
  - 灰度/回滚：滚动发布、蓝绿发布（了解）
- **验收产出**
  - 合并即触发：自动化测试 + 构建镜像 + 部署到测试环境

### 4.5 Kubernetes（选修：理解概念 + 逐步落地）

- **核心概念**
  - Pod、Deployment、Service、Ingress、ConfigMap、Secret
  - HPA（自动扩缩容）、资源配额与探针（liveness/readiness）
- **验收产出**
  - 将一套服务以 Deployment 方式运行，并配置 Ingress 暴露

## 5. 可观测性与运维（核心）

### 5.1 日志（Logging）

- **重要组成部分**
  - 结构化日志（JSON）、分级（info/warn/error）
  - 关联字段：requestId/traceId、userId、业务关键字段
  - 日志脱敏：手机号/邮箱/证件号、token、密码等不落日志
- **验收产出**
  - 能通过日志快速定位一次线上故障：请求链路、错误原因、影响面

### 5.2 指标（Metrics）

- **核心概念**
  - RED/USE 方法：请求率、错误率、延迟；资源使用
  - SLI/SLO（了解）：定义可用性目标与告警阈值
- **验收产出**
  - Dashboard：QPS、P95、错误率、CPU/内存、缓存命中率

### 5.3 链路追踪（Tracing）

- **核心概念**
  - Trace/Span、上下文传播、采样策略
  - 跨服务调用定位瓶颈（数据库、RPC、缓存）
- **验收产出**
  - 一次请求从网关到数据库的端到端耗时拆解

### 5.4 告警与应急

- **重要组成部分**
  - 告警分级：P0/P1/P2；噪声控制（抑制/聚合）
  - Runbook：定位步骤、回滚/降级策略
  - 事故复盘：时间线、根因、行动项
- **验收产出**
  - 一份可执行的故障手册 + 复盘模板（团队可复用）

## 6. 测试体系（核心）

### 6.1 单元测试（Unit Test）

- **重要组成部分**
  - Service/Domain 层测试为主，Controller 层薄测试
  - Mock 外部依赖（DB/HTTP/MQ）
  - 覆盖核心分支与边界条件
- **验收产出**
  - 核心业务逻辑覆盖率可衡量、回归成本显著降低

### 6.2 集成测试（Integration Test）

- **重要组成部分**
  - 使用真实数据库/Redis（容器化）验证事务与 SQL 正确性
  - 迁移脚本在测试环境可重复执行
- **验收产出**
  - “一键跑通”集成测试：从 API 到 DB 的关键链路

### 6.3 API 契约与回归

- **重要组成部分**
  - OpenAPI 作为契约：请求/响应 schema 固化
  - 接口回归集合：对关键接口定期运行
- **验收产出**
  - 每次发布前自动跑接口回归，避免破坏兼容性

### 6.4 E2E（端到端）

- **重要组成部分**
  - 覆盖核心用户路径：注册/登录/下单/支付（或业务核心流程）
  - 数据准备与清理策略、稳定性（避免随机失败）
- **验收产出**
  - 关键流程稳定 E2E，用于发版验收

  - 交付：上线、监控、回滚、演练

  - 一份小系统设计文档（含：数据模型、API、异常场景、监控指标）

### 9.2 高并发与性能（按需深入）

- **重要组成部分**
  - 读多写少：缓存、CDN、分页与索引
  - 写多：队列削峰、批处理、异步化、分库分表（了解）
  - 热点治理：热点 key、热点行、热点接口
- **验收产出**
  - 针对热点接口的优化前后对比：P95、错误率、资源消耗

### 9.3 一致性与可靠性（工程落地）

- **重要组成部分**
  - 幂等 + 重试 + 超时（组合拳）
  - 补偿机制：失败回滚、对账、人工处理入口
- **验收产出**
  - 关键链路具备失败恢复能力，且可观测（日志/告警/回放）

## 11. 实战项目建议（把知识“串起来”）

建议用一个中等复杂度项目完成闭环（任选其一）：

- **多租户后台系统（ToB）**
  - 组织/用户/角色权限（RBAC）
  - 数据隔离（租户字段隔离起步）
  - 审计日志、导入导出、对象存储
- **电商/交易链路（ToC）**
  - 商品/库存/订单/支付（支付可做模拟）
  - 并发扣库存（事务 + 锁/乐观锁）
  - MQ 异步：通知、发券、对账
- **内容/社区系统（中等并发）**
  - 搜索（ES 可选）、推荐（简单规则）
  - 缓存与热点治理、内容审核流程（可简化）

最小交付标准（建议写成验收清单）：

- API 文档完整（OpenAPI）且可回归测试
- 鉴权与权限可配置
- 数据库迁移可重复执行
- Docker 一键启动（含依赖）
- 具备日志/指标/告警（至少基础版本）
- 具备单测 + 集成测试（覆盖关键链路）
